# Forge V1.1 ‚Äî Workspace Sidebar, Projects, Session Lifecycle, Reasoning Effort

Approved 2026-07-10. Extends V1 (spec: `2026-07-10-forge-agent-workspace-design.md`, both phases merged to main). The 2a visual language (tokens, hairlines, type scale in `web/src/styles/tokens.css`) is the styling authority; V1.1 adds a sidebar-based information architecture on top of it.

## Goals and non-goals

**Goals**
1. A full-height left **sidebar**: projects with their sessions, ad-hoc sessions, archived sessions, new-project/new-session entry points. Replaces the top-bar session tabs.
2. **Projects**: named folder + defaults (model, autonomy, reasoning effort), persisted server-side with REST CRUD.
3. **Session lifecycle**: archive/unarchive (kept on disk, restorable) and permanent delete.
4. **Reasoning effort**: per-session `default | low | medium | high`, seeded by project defaults, passed to CLIProxyAPI as `reasoning_effort` (verified accepted by both GPT and Claude model families through the proxy).

**Non-goals:** visible model/autonomy buttons (palette-only stays), per-project skills, drawer changes, sidebar drag-reorder, multi-window.

## Engine

### Projects store (`server/forge/store/projects.py`)

- File: `~/.forge/projects.json` ‚Äî a JSON array of `{id, name, cwd, default_model, default_autonomy, default_effort}`. `id` = `uuid4().hex[:8]`. Defaults fall back to `ForgeConfig` values when omitted.
- `ProjectStore(home)` with `list() / get(pid) / create(...) / update(pid, ...) / delete(pid)`; loads at init, saves on every mutation (atomic write: tmp + rename).
- REST:
  - `GET /api/projects` ‚Üí `Project[]`
  - `POST /api/projects` `{name, cwd, default_model?, default_autonomy?, default_effort?}` ‚Üí `Project` (400 if cwd is not an existing directory)
  - `PATCH /api/projects/{pid}` (any subset of fields) ‚Üí `Project`; 404 unknown
  - `DELETE /api/projects/{pid}` ‚Üí `{}`; 404 unknown. Deleting a project never touches its sessions ‚Äî they simply render under Ad-hoc afterwards.

### Session ‚Üî project, archive, delete

- `SessionCreated` gains `project_id: str | None = None` (old logs parse unchanged). `SessionMeta` gains `project_id: str | None`, `archived: bool = False`, `effort: Effort = "default"`.
- `CreateSession` schema gains `project_id` and `effort`. Server-side resolution on create: explicit field > project default (when `project_id` given) > config default. Unknown `project_id` ‚Üí 400.
- New durable events: `SessionArchived`, `SessionUnarchived` (no payload beyond the base), `EffortChanged(effort)`. All applied in `_replay_meta`.
- `POST /api/sessions/{sid}/archive` ‚Üí 409 while a run is active, else emits `session_archived`, meta.archived=True. `POST .../unarchive` mirror.
- `DELETE /api/sessions/{sid}` ‚Üí 409 while running; removes the actor from the manager (actors dict + creation order), `shutil.rmtree` of `~/.forge/sessions/{sid}`, then broadcasts an **ephemeral** `SessionDeleted(session_id, seq=0)` on the bus so connected clients drop it live (it cannot be a durable event ‚Äî its log is gone). Rehydration naturally never sees it again.
- `GET /api/sessions` includes the new meta fields; archived sessions are included (UI segregates them).
- `GET /api/recent_dirs` ‚Üí up to 10 distinct session cwds, most recently created first (derived from live metas; no new storage). Feeds the ad-hoc dialog's suggestions.

### Reasoning effort

- `Effort = Literal["default", "low", "medium", "high"]`.
- `LLMClient.complete` gains keyword param `effort: str = "default"`. `OpenAILLM` adds `reasoning_effort=effort` to the request only when `effort != "default"`. `FakeLLM` records the received effort per call (test seam).
- `SessionActor` passes `self.meta.effort` on every loop completion and on the compaction summarizer call. `set_effort(effort)` command mirrors `set_autonomy` (meta + `EffortChanged` emit). `POST /api/sessions/{sid}/effort` `{effort}` ‚Üí 400 on invalid value.

### Protocol

`Event` union grows by `session_archived`, `session_unarchived`, `effort_changed`; `SessionMeta` grows three fields; new ephemeral `SessionDeleted`. Regenerate `web/src/protocol/generated.ts` via `pnpm gen:protocol` (add `session_deleted` to the export bundle and the contract test's literal list).

## Web

### Layout

Frame becomes a row: **Sidebar** (full height, w 232px, bg `--bg-drawer`, border-right `--hair-2`) + right column (TopBar + main). TopBar loses the session-tab segment; keeps brand, queue pill (all-sessions queued count), active cwd. ChatStream/Composer/DetailDrawer unchanged.

### Sidebar

Top-to-bottom:
1. `PROJECTS` label (10.5px mono, `--text-ghost`, letter-spacing .08em).
2. One collapsible section per project (from `/api/projects`): header row `‚ñæ/‚ñ∏ {name}` (12px/500; chevron toggles, collapsed state local to the UI); under it the project's sessions and a ghost `Ôºã new session` row (creates immediately with project defaults, activates it).
3. `AD-HOC` section: sessions with no/unknown `project_id`; its `Ôºã new session‚Ä¶` opens the New Session dialog.
4. `Ôºã New project‚Ä¶` row ‚Üí New Project dialog.
5. `ARCHIVED (n)` collapsed section at the bottom (rows muted; row actions: unarchive, delete).

Session row: 6px status dot (same rules as V1 tabs: active accent+glow / busy `--warn-dot` / idle `#3d3d47`), name ellipsized, hover reveals right-side actions ‚Äî active/idle rows: archive (`‚å´`-style ghost icon button); archived rows: unarchive + delete. Delete opens a small confirm dialog (danger-styled Delete button + Cancel). Row click activates the session (archived rows activate a read-only view ‚Äî the stream is just a projection; the composer is disabled for archived sessions with placeholder "Archived ‚Äî unarchive to continue", so nothing can post to one without an explicit unarchive). Rename stays available via PATCH later; not surfaced as a V1.1 control.

### Dialogs (shared modal styling: centered card, `--bg-raised-1`, radius `--r-card`, hairline, shadow like the composer)

- **New Session** (ad-hoc): mono path input with suggestions from `/api/recent_dirs` (click to fill), selects for model (config list) / autonomy / effort pre-filled from config defaults, `Start session` accent button. Invalid path ‚Üí inline error from the 400.
- **New Project**: name input + everything above; `Create project`. Creating does not auto-start a session.

### State & wiring

- Store: `projects: Project[]` (loaded in `hydrate`), session state gains `projectId/archived/effort` (seeded from meta, updated by the new events through the reducer); `applyEvent` intercepts `session_deleted` and removes the session (store-level ‚Äî the reducer stays per-session); actions `createProject/updateProject/deleteProject`, `archive/unarchive/deleteSession`, `setEffort`, `newSessionInProject(pid)`, `newAdhocSession(opts)`.
- Reducer: `session_created` records `projectId`; `session_archived/unarchived` flip `archived`; `effort_changed` sets `effort`.
- Model pill: `{model display} ¬∑ {effort} ¬∑ {autonomy}` with effort omitted when `default`. Palette gains `/effort` (submenu default/low/medium/high). `/new` now opens the New Session dialog.

## Error handling

Archive/delete race with a run ‚Üí 409 surfaced as a transient inline error near the row. Deleting the active session activates the next visible session (or empty state: centered ghost "No session ‚Äî create one from the sidebar"). Project CRUD failures ‚Üí inline dialog errors. `session_deleted` for the active session behaves like local delete.

## Testing

- **Engine (pytest, FakeLLM/TestClient):** projects CRUD + persistence round-trip + 400s; create-session default resolution (explicit > project > config); archive/unarchive events + replay + 409-while-running; delete (409 running, dir gone, 404 after, ephemeral broadcast observed on WS); effort event + replay + FakeLLM receives `reasoning_effort` (and not when default).
- **Web (vitest):** reducer new events; store `session_deleted` removal + active-session fallback; Sidebar rendering/grouping/collapse/actions; dialogs (create paths, validation error); `/effort` palette; pill with/without effort.
