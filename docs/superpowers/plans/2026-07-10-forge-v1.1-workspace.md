# Forge V1.1 — Workspace Sidebar, Projects, Lifecycle, Effort — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement the approved V1.1 spec (`docs/superpowers/specs/2026-07-10-forge-v1.1-workspace-design.md`): a full-height workspace sidebar (projects → sessions, ad-hoc, archived), project CRUD with defaults, session archive/unarchive/delete, and per-session reasoning effort passed through to CLIProxyAPI.

**Architecture:** V1 conventions carry over unchanged — the engine stays event-sourced (new facts become durable events replayed by `_replay_meta`; the one thing that cannot be a durable event, session deletion, is an ephemeral bus broadcast), the UI stays a pure projection (reducer handles the new events; commands are fire-and-forget REST). Projects are server-side config-like state (`~/.forge/projects.json` via a small store), not events. Tasks 1–5 are Python, 6–12 web.

**Tech Stack:** unchanged from V1 (FastAPI/pydantic/pytest in `server/`; React/Vite/TS/Zustand/vitest in `web/`).

## Global Constraints

- **Engine:** run `uv run pytest -q` / `uv run ruff check .` from `server/`; ruff line-length 100; tests use the FakeLLM + TestClient patterns in `server/tests/` (fixture `make_client(script=None, delay=0.0)` in `test_api_extras.py`); pristine output; TDD per task.
- **Web:** run from `web/`: `pnpm test` (vitest, `globals: false` — explicit imports), `pnpm build` (tsc -b + vite). TS strict, no `any` in `web/src` (tests may cast). **`erasableSyntaxOnly` is on: constructor parameter properties (`constructor(public x: string)`) do not compile — use explicit fields.** `web/src/test/setup.ts` already registers `afterEach(cleanup)` + jest-dom. Store reset pattern in tests: `useForge.setState(useForge.getInitialState(), true)`.
- **Generated protocol types have optional fields** for anything with a pydantic default (`seq?`, `project_id?`, `archived?`, `effort?`, …) — consume with `??` fallbacks, never non-null assertions on wire data.
- **Styling:** reuse the existing custom properties in `web/src/styles/tokens.css` — no new hex values except where this plan writes one explicitly. New chrome (sidebar, modals) follows the 2a language: hairlines by elevation, radius `--r-card`/`--r-btn-lg`, mono for paths/meta, type scale 10.5–13.5px.
- **Effort values:** exactly `default | low | medium | high`; `default` means the `reasoning_effort` param is omitted entirely.
- **Event-sourced UI:** components never mutate session state directly; archive/effort/etc. round-trip through REST → engine event → WS → reducer. The single exception is `session_deleted` handling (store removes the entry).
- **Commits:** small, conventional prefixes (`feat:`, `fix:`, `test:`, `chore:`).

## Protocol additions (summary the tasks implement)

Durable events: `session_archived`, `session_unarchived`, `effort_changed(effort)`; `session_created` gains `project_id: str | None = None` and `effort: Effort = "default"`. Ephemeral: `session_deleted` (seq 0, only `session_id` + type). `SessionMeta` gains `project_id`, `archived`, `effort`. New REST: `GET/POST /api/projects`, `PATCH/DELETE /api/projects/{pid}`, `GET /api/recent_dirs`, `POST /api/sessions/{sid}/archive|unarchive|effort`, `DELETE /api/sessions/{sid}`. `CreateSession` gains `project_id`/`effort` with server-side resolution explicit > project default > config default.

```python
from __future__ import annotations

import json
import os
from pathlib import Path
from uuid import uuid4

from pydantic import BaseModel


class Project(BaseModel):
    id: str
    name: str
    cwd: str
    default_model: str = ""     # "" = unset → config default at session create
    default_autonomy: str = ""
    default_effort: str = ""


class ProjectStore:
    def __init__(self, home: Path):
        self._path = home / "projects.json"
        self._projects: list[Project] = []
        if self._path.exists():
            self._projects = [Project.model_validate(p)
                              for p in json.loads(self._path.read_text())]

    def _save(self) -> None:
        tmp = self._path.with_suffix(".json.tmp")
        tmp.write_text(json.dumps([p.model_dump() for p in self._projects], indent=2))
        os.replace(tmp, self._path)  # atomic on POSIX

    def list(self) -> list[Project]:
        return list(self._projects)

    def get(self, pid: str) -> Project | None:
        return next((p for p in self._projects if p.id == pid), None)

    def create(self, name: str, cwd: str, default_model: str = "",
               default_autonomy: str = "", default_effort: str = "") -> Project:
        p = Project(id=uuid4().hex[:8], name=name, cwd=cwd,
                    default_model=default_model, default_autonomy=default_autonomy,
                    default_effort=default_effort)
        self._projects.append(p)
        self._save()
        return p

    def update(self, pid: str, fields: dict) -> Project | None:
        p = self.get(pid)
        if p is None:
            return None
        updated = p.model_copy(update=fields)
        self._projects[self._projects.index(p)] = updated
        self._save()
        return updated

    def delete(self, pid: str) -> bool:
        p = self.get(pid)
        if p is None:
            return False
        self._projects.remove(p)
        self._save()
        return True
```

```tsx
  title,

  children: ReactNode

      <div className={s.card} role="dialog" aria-label={title}>
        <div className={s.title}>{title}</div>
        {children}
      </div>
    </div>
  )
```

```tsx
import { useState } from 'react'
import type { Project } from '../protocol'
import { useForge, type SessionState } from '../state/store'
import ConfirmDialog from './ConfirmDialog'
import s from './Sidebar.module.css'

export function groupSessions(
  projects: Project[],
  order: string[],
  sessions: Record<string, SessionState>,
): { byProject: Record<string, string[]>; adhoc: string[]; archived: string[] } {
  const ids = new Set(projects.map(p => p.id))
  const byProject: Record<string, string[]> = Object.fromEntries(projects.map(p => [p.id, []]))
  const adhoc: string[] = []
  const archived: string[] = []
  for (const id of order) {
    const { projectId, archived: isArchived } = sessions[id].stream
    if (isArchived) archived.push(id)
    else if (projectId && ids.has(projectId)) byProject[projectId].push(id)
    else adhoc.push(id)
  }
  return { byProject, adhoc, archived }
}

function Dot({ active, busy }: { active: boolean; busy: boolean }) {
  return <span className={s.dot} data-state={active ? 'active' : busy ? 'busy' : 'idle'} />
}

export default function Sidebar() {
  const projects = useForge(st => st.projects)
  const order = useForge(st => st.order)

  const setActive = useForge(st => st.setActive)
  const openDialog = useForge(st => st.openDialog)

  const archiveSession = useForge(st => st.archiveSession)
  const unarchiveSession = useForge(st => st.unarchiveSession)

    setCollapsed(prev => {
      const next = new Set(prev)
      if (next.has(key)) next.delete(key)
      else next.add(key)
      return next
    })

      ? 'Session is running — cancel it first'
      : 'Action failed'
    setRowError({ id, msg })

  }

                <button className={s.action} aria-label={`Unarchive ${st.name}`}
                        title="Unarchive"
                        onClick={() => void unarchiveSession(id).catch(failSoft(id))}>⤴</button>
                <button className={s.actionDanger} aria-label={`Delete ${st.name}`}
                        title="Delete" onClick={() => setConfirmDelete(id)}>✕</button>
              </>
            ) : (
              <button className={s.action} aria-label={`Archive ${st.name}`}
                      title="Archive"
                      onClick={() => void archiveSession(id).catch(failSoft(id))}>⌫</button>
            )}
          </span>
        </div>
        {rowError?.id === id && <div className={s.rowError}>{rowError.msg}</div>}
      </div>
    )
  }

  return (
    <nav className={s.sidebar}>
      <div className={s.label}>PROJECTS</div>

      {projects.map(p => (
        <section key={p.id}>
          <button className={s.sectionHeader} onClick={() => toggle(p.id)}>
            <span className={s.chevron}>{collapsed.has(p.id) ? '▸' : '▾'}</span>
            {p.name}
          </button>
          {!collapsed.has(p.id) && (
            <>
              {byProject[p.id].map(id => row(id, false))}
              <button className={s.plusRow} aria-label={`New session in ${p.name}`}
                      onClick={() => void newSessionInProject(p.id).catch(failSoft(p.id))}>
                ＋ new session
              </button>
            </>
          )}
        </section>
      ))}

      <div className={s.label}>AD-HOC</div>
      {adhoc.map(id => row(id, false))}
      <button className={s.plusRow} aria-label="New ad-hoc session"
              onClick={() => openDialog('new-session')}>
        ＋ new session…
      </button>

      <button className={s.newProject} aria-label="New project"
              onClick={() => openDialog('new-project')}>
        ＋ New project…
      </button>

      {archived.length > 0 && (
        <section className={s.archivedSection}>
          <button className={s.sectionHeader} onClick={() => toggle('__archived__')}>
            <span className={s.chevron}>{collapsed.has('__archived__') ? '▸' : '▾'}</span>
            ARCHIVED ({archived.length})
          </button>
          {!collapsed.has('__archived__') && archived.map(id => row(id, true))}
        </section>
      )}

      {confirmDelete && (
        <ConfirmDialog
          title="Delete session"
          body={`This will permanently delete "${sessions[confirmDelete].stream.name}" and its history.`}
          confirmLabel="Delete"
          onCancel={() => setConfirmDelete(null)}
          onConfirm={() => {
            const id = confirmDelete
            setConfirmDelete(null)
            void deleteSession(id).catch(failSoft(id))
          }}
        />
      )}
    </nav>
  )
}
```
